# 🚀 Event-Driven Data Platform (GCP)

This project is a serverless event-driven data pipeline built on Google Cloud Platform (GCP). It processes real-time events, stores them in a data warehouse, and enables analytics through BI dashboards.

---

![Architecture](docs/architecture.png)

---

## ⚙️ Overview

The system is designed to ingest, process, and analyze event data in near real-time using a fully serverless architecture.

---

## 🧩 Components

### 1. Event Processing (Cloud Run)
- Google Cloud Run is used as the compute layer.
- A Python service receives HTTP events in JSON format.
- The payload is validated and transformed into a structured schema.
- Data is streamed into BigQuery.

### 2. Data Warehouse (BigQuery)
- Google BigQuery stores all processed events.
- Acts as the single source of truth for analytics.
- Enables fast SQL-based querying over large datasets.

### 3. Analytics Layer (Looker)
- Looker dashboards are built on top of BigQuery.
- Provides business KPIs and real-time metrics.
- Used for reporting and data visualization.

  --allow-unauthenticated

---

## 🔁 Data Flow

1. An event is generated by an external system.
2. The event is sent via HTTP to Cloud Run.
3. The Python service processes and transforms the event.
4. The structured data is inserted into BigQuery.
5. Looker queries BigQuery to generate dashboards and metrics.
